Rewrite Configuration:

For our MVC framework to work with clean URLs the web server must define custom Rewrite rules

With Apache the Rewrite configuration is defined by .htaccess file on project root folder. There is no need to modify any other setting.

With Nginx Rewrite configuration is defined on the site configuration file (of the server) - usually on: /etc/nginx/sites-available/site_name.conf. In this file add a section similar to this one (change the /myfolder/ path to the exact subdomain or domain of your site):

location /myfolder/ {
	try_files $uri $uri/ /myfolder/index.php?$query_string;
}

Note:

If you are using the provided homestead.yaml file and configuring your Vagrant Homestead accordingly, then the Nginx configuration is pre-defined, as long as the sites of both examples (09.complete and 10.ourframework) are:
